# 一次性配置说明（只需填 2 处）

## 第一处：`cloud-config.js`（3 个公开值）

打开项目里的 **`cloud-config.js`**，填入：

```javascript
window.CARD_CLOUD = {
  supabaseUrl: "https://xxxx.supabase.co",           // ①
  supabaseAnonKey: "eyJhbGci...",                    // ②
  saveFunctionUrl: "https://xxxx.supabase.co/functions/v1/save-card",  // ③
};
```

| 序号 | 从哪复制 |
|------|----------|
| ① | Supabase → **Settings** → **API** → **Project URL** |
| ② | 同上 → **anon public**（不是 service_role） |
| ③ | 部署 `save-card` 函数后的地址（见下方一次性初始化） |

> 这三项可以提交到 GitHub，**不是私密 Token**。

---

## 第二处：Supabase 服务端密钥（私密，只填一次）

在 Supabase 控制台或 CLI 设置，**不要写进任何 .js 文件**：

```
CARD_EDIT_PASSWORD = 763560
```

与 `data.js` 里的 `editPassword` 保持一致。

CLI 命令：

```powershell
supabase secrets set CARD_EDIT_PASSWORD=763560
```

---

## 一次性初始化 Supabase（首次必做）

1. [supabase.com](https://supabase.com) 注册 → **New project**
2. **SQL Editor** → 粘贴运行 `supabase/schema.sql` 全部内容
3. 安装 CLI 后执行：

```powershell
cd D:\Code\person_web
supabase login
supabase link --project-ref 你的项目ref
supabase secrets set CARD_EDIT_PASSWORD=763560
supabase functions deploy save-card
```

4. 把 ①②③ 填进 `cloud-config.js`
5. 推送：

```powershell
git add cloud-config.js
git commit -m "配置云端同步"
git push
```

---

## 不需要填 Token 的地方

| ❌ 不要填 | 原因 |
|----------|------|
| GitHub Personal Access Token | 不能放前端，GitHub 会拦截 |
| Supabase **service_role** 密钥 | 只能在服务端，Edge Function 自动使用 |
| `data.js` | 只放名片默认内容和编辑密码 |

2. 双击「个人名片」→ 密码 `763560` → 编辑 → 保存  
3. 其他设备刷新即可看到相同内容
