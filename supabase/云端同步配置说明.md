# 云端同步配置说明（无需在前端放 Token）

GitHub Token **不能**写进网页代码（会被公开）。改用 **Supabase 免费云数据库**：

- 前端只放**公开的**读取密钥（只能读，不能改）
- **保存密码**放在 Supabase 服务端（Edge Function 密钥里）
- 别人看不到你的写入权限

---

## 第一步：注册 Supabase

1. 打开 [https://supabase.com](https://supabase.com) 注册并登录  
2. **New project** → 取名如 `person-card` → 设置数据库密码 → 创建  

---

## 第二步：建表

1. 左侧 **SQL Editor** → **New query**  
2. 复制项目里 `supabase/schema.sql` 的全部内容，粘贴并 **Run**

---

## 第三步：部署保存接口（Edge Function）

1. 安装 [Supabase CLI](https://supabase.com/docs/guides/cli)（或使用 Dashboard 在线创建 Function）  
2. 在项目根目录登录并关联项目：

```powershell
cd D:\Code\person_web
supabase login
supabase link --project-ref 你的项目ref
```

项目 ref 在 Supabase 控制台 URL 里：`https://supabase.com/dashboard/project/这里就是ref`

3. 设置**服务端密码**（与 `data.js` 里 `editPassword` 一致，例如 `763560`）：

```powershell
supabase secrets set CARD_EDIT_PASSWORD=763560
```

4. 部署函数：

```powershell
supabase functions deploy save-card
```

5. 记下函数地址，格式如下：

```
https://你的项目ref.supabase.co/functions/v1/save-card
```

---

## 第四步：填写 `data.js`（你要手动粘贴的位置）

打开 **`data.js`**，找到 **`cloudSync`**，填入三项（**不要填 Token**）：

```javascript
cloudSync: {
  // ① Supabase 项目 URL（Settings → API → Project URL）
  supabaseUrl: "https://xxxxxxxx.supabase.co",

  // ② 匿名公钥 anon public（Settings → API → anon public）
  supabaseAnonKey: "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",

  // ③ 上一步部署的 Edge Function 地址
  saveFunctionUrl: "https://xxxxxxxx.supabase.co/functions/v1/save-card",
},
```

### 在 Supabase 控制台哪里找 ① 和 ②？

**Settings** → **API**：

| 字段 | 复制项 |
|------|--------|
| `supabaseUrl` | **Project URL** |
| `supabaseAnonKey` | **anon public**（可公开，只能读名片） |

### ③ 填什么？

部署 `save-card` 函数后得到的完整 URL。

---

## 第五步：推送代码

```powershell
git add data.js
git commit -m "配置 Supabase 云端同步"
git push
```

---

## 使用方式

1. 任意设备打开名片页 → 自动从 Supabase **读取**最新数据  
2. 双击「个人名片」→ 输入密码 → 编辑 → **保存修改**  
3. 保存时会调用 Edge Function，在**服务端**校验密码后写入  
4. 其他设备**刷新页面**即可看到相同内容  

---

## 安全说明

| 放在哪里 | 内容 | 是否可公开 |
|----------|------|------------|
| `data.js` | `supabaseAnonKey` | ✅ 可以（仅只读） |
| `data.js` | `editPassword` | ⚠️ 网页里可见，仅防误改 |
| Supabase Secrets | `CARD_EDIT_PASSWORD` | ✅ 服务端私密 |
| Supabase 后台 | `service_role` 密钥 | ❌ 永远不要写进前端 |

**切勿**把 GitHub Token 或 Supabase 的 **service_role** 密钥写进 `data.js` 或提交到 GitHub。
